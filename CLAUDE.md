# Design Doc Formatting

When writing design docs or markdown reference files:

- 80-column line wrap
- Table of Contents with `- [Section](#lowercase-hyphen-anchor)`
- 4-space indent nesting
- Fragment bullets: no trailing period
- Sentence bullets: trailing period
- Use ASCII diagrams over prose when showing architecture
- Headings: sentence case
- File naming: lowercase-hyphen.md

# Git Commits

- Do not add Claude attribution (no `Co-Authored-By: Claude` trailers, no
  "Generated with Claude Code" lines) when committing code.
